成功标准 2.4.11 焦点外观（最低）

了解焦点外观（最小） |
如何满足焦点外观（最低）

(AA级)

[新的]

当用户界面组件接收键盘焦点时，以下所有情况都为真：

  • 对比区域：对焦指示器中有一个 区域，其在对焦和未对焦状态下的颜色之间的对比度至少为 3:1。
  • 最小面积：对比面积至少与：
    • 轮廓：未聚焦组件的 1 CSS 像素厚周长的区域，或
    • 形状：沿着未聚焦组件的最小边界框的最短边的 4 CSS 像素粗线的区域，并且不小于 2 CSS 像素。
  • 相邻对比度：对比区域与聚焦组件中相邻颜色的对比度也至少为3:1，或者对比区域的厚度至少为2个CSS像素。
  • 未完全隐藏：具有焦点的项目并未完全被作者创建的内容隐藏。
笔记

具有图案或渐变的键盘对焦指示器可能会有不满足 3:1 对比度变化的部分，只要等于最小值的区域满足对比度即可。

笔记

如果组件的可见边界小于命中区域，或者组件的大小不可用，则可以从可见边界中取最小区域。

编者注

工作组对关于最小面积度量的反馈感兴趣，以及是否存在可见指标被措辞捕获的异常情况。

成功标准 2.4.12 焦点外观（增强）

了解焦点外观（增强） |
如何满足焦点外观（增强）

（AAA级）

[新的]

当用户界面组件具有键盘焦点时，以下所有情况都为真：

  • 对比区域：对焦指示器中有一个区域，其在对焦和未对焦状态下的颜色之间的对比度至少为 4.5:1。
  • 最小面积：对比区域至少是未聚焦组件的 1 CSS 像素 周长面积的两倍；
  • 未被遮挡：作者创建的内容不会隐藏焦点指示器的任何部分。

成功标准 2.5.7 拖动动作

了解拖动动作 |
如何满足拖动动作

(AA级)

[新的]

所有使用拖动动作进行操作的功能都可以通过单个指针实现而无需拖动，除非拖动是必不可少的。

笔记

此要求适用于解释指针操作的 Web 内容（即这不适用于操作用户代理或辅助技术所需的操作）。

编者注

是否有帮助行动不便人士的辅助技术？该小组希望就 AT 与作者责任之间的边界提供反馈。

成功标准 2.5.8 目标尺寸（最小）

了解目标尺寸（最小） |
如何满足目标尺寸（最小）

(AA级)

[新的]

目标的面积至少为 24 x 24 CSS 像素，以下情况除外：

  • 间距：所述目标偏移量是至少24个CSS像素的每一个的相邻目标;
  • Inline：目标在一个句子或一段文本中；
  • 必不可少：目标的特定呈现对于所传达的信息至关重要。
笔记

考虑到成功标准的目的，允许基于目标内的位置在空间上选择值的目标被视为一个目标。示例包括具有粒度值的滑块、显示颜色渐变的颜色选择器或放置光标的可编辑区域。

编者注

描述内联链接时是否存在国际化问题？

编者注

弹出内容重叠目标是否存在触发失败的问题？

成功标准 3.2.6 始终如一的帮助

了解一致的帮助 |
如何获得一致的帮助

(A级)

[新的]

对于每一个网页一个内一套网页，它提供了寻求帮助的下列方式之一或更多，获得的帮助下至少一种形式被包含在每个页面上相同的相对顺序：

  • 人员联系方式；
  • 人体接触机制；
  • 自助选项；
  • 完全自动化的联系机制。
笔记

可以直接在页面上提供对帮助机制的访问，也可以通过指向包含信息的不同页面的直接链接来提供。

成功标准 3.2.7 可视控件

了解可见控件 |
如何满足可视化控制

(AA级)

[新的]

在接收指针悬停或键盘焦点触发用户界面组件可见的情况下，识别用户界面组件可用所需的信息是可见的，除非：

  • 识别用户界面组件所需的信息可通过在同一页面或多步骤过程中的不同步骤可见的等效组件获得，无需指针悬停或键盘焦点；
  • 该组件专门用于增强键盘导航的体验；
  • 有一种机制可以使信息持久可见；
  • 隐藏识别组件所需的信息至关重要。
笔记

用户界面组件可以通过其他可见组件使用，例如子菜单、编辑按钮、选项卡或媒体缩略图。

编者注

工作组对有关是否存在由用户代理确定的不应在范围内的组件的反馈感兴趣。

成功标准 3.3.7 无障碍认证

了解无障碍身份验证 |
如何满足可访问的身份验证

(A级)

[新的]

对于依赖认知功能测试的认证过程中的每一步，至少有一种不依赖于认知功能测试的其他认证方法可用，或者有一种机制可以帮助用户完成认知功能测试。

笔记

机制示例包括：1) 支持密码管理员输入密码以解决记忆认知功能测试，以及 2) 复制和粘贴以帮助解决转录认知功能测试。

成功标准 3.3.8 冗余入口

了解冗余入口 |
如何满足冗余入口

(A级)

[新的]

需要在同一进程和同一用户会话中再次输入的先前由用户输入或提供给用户的信息是：

  • 自动填充，或
  • 可供用户选择。

除非：

  • 重新输入信息是必不可少的，
  • 需要该信息以确保内容的安全性，或
  • 以前输入的信息不再有效。
编者注

存储数据是否存在问题，以便用户可以在后续步骤中访问它？

编者注

是否需要比必要的更广泛的例外？例如，用于强制或要求的信息重新输入。
